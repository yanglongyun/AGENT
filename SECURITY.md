# 安全政策

## 支持范围

项目当前尚未建立长期支持分支，安全修复仅针对 `main` 分支的最新代码。请在报告前确认问题仍可在最新版本中复现。

## 私下报告漏洞

请不要在公开 Issue 中提交漏洞细节、利用代码、密钥或用户数据。

1. 通过 Gitee 私信联系 [@realuckyang](https://gitee.com/realuckyang)。
2. 附上受影响的版本或提交、复现步骤、可观察影响和建议修复。
3. 如无法私信，请创建一个仅包含“需要私下报告安全问题”及联系方式的 Issue，不要公布技术细节。

维护者会尽快确认收到报告、评估影响并协调修复与披露时间。在修复发布前，请给予合理处理时间并保持细节私密。

## 项目安全边界

- Agent Chat 可在本机无沙箱执行 shell 命令，应只与可信模型及可信输入一起使用。
- 服务器默认用于本机访问，不应在没有额外身份验证、TLS 和访问控制的情况下暴露到公网。
- 浏览器扩展拥有高权限，可访问页面、标签和调试协议；不使用时应断开或禁用。
- `.env`、API Key、`data/` 和 `uploads/` 不得提交到仓库。
